# 介绍

db-ferry 是一个数据搬运工具，它可以帮助你把数据从一个数据库迁移到另一个数据库。就像一个"数据渡轮"，把数据从"A码头"运送到"B码头"。

## 它能做什么？

- 支持 Oracle、MySQL、PostgreSQL、SQL Server、SQLite 和 DuckDB 之间的数据迁移
- 自动创建目标表结构（无需手动建表）
- 支持批量数据迁移，效率高
- 提供进度条，实时查看迁移状态
- 迁移完成后自动创建索引，加快查询速度

## 适合谁用？

- **数据分析师**：需要将生产库数据导入本地分析
- **开发人员**：需要在不同环境同步数据
- **业务人员**：需要定期备份或迁移业务数据

## 核心特性

- **多数据库支持**：Oracle、MySQL、PostgreSQL、SQL Server、SQLite、DuckDB
- **声明式配置**：一份 `task.toml` 定义所有数据库连接和迁移任务
- **自动建表**：根据源数据自动创建目标表结构
- **批量高效**：默认每批 1000 条数据，速度快
- **DAG 并行调度**：无依赖任务自动并行执行
- **断点续传**：支持 `resume_key` 和状态文件，中断后可继续
- **数据校验**：行数、哈希、抽样三级校验
- **死信队列**：失败行单独持久化，不导致整个任务失败
- **PII 脱敏**：内置 8 种脱敏规则
- **定时调度**：内置 cron 支持自动化同步
